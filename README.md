# Otchet V666 (Bot Lion Otchet)

**Репозиторий:** https://github.com/preudd/otchetV666

Telegram-бот вечернего отчёта — запись Excel в Google Таблицу.
Бот обрабатывает Excel с экспортом чеков и записывает данные в **Google Таблицу**. Отправьте боту файл — он заполнит таблицу (маппинг ячеек настраивается под ваш шаблон).

## Установка

1. Установите зависимости:
   ```
   pip install -r requirements.txt
   ```

2. Создайте бота в Telegram: [@BotFather](https://t.me/BotFather), команда `/newbot`.

3. Скопируйте `config.example.py` в `config.py` и укажите:
   - `BOT_TOKEN` — токен от BotFather
   - `GOOGLE_SPREADSHEET_ID` — ID вашей таблицы из URL
   - `GOOGLE_SERVICE_ACCOUNT_FILE` — путь к JSON-ключу сервисного аккаунта

4. Создайте в таблице листы по месяцам: `январь`, `февраль`, …, `декабрь` (или переименуйте текущий `Лист1` в `июль`).

## Подключение Google Таблицы

Таблица должна **уже существовать** — бот её не создаёт.

1. В [Google Cloud Console](https://console.cloud.google.com/) создайте проект.
2. Включите **Google Sheets API**.
3. Создайте **сервисный аккаунт** → ключ → JSON. Сохраните как `service_account.json` в папке проекта.
4. Откройте вашу Google Таблицу → **Поделиться** → добавьте email сервисного аккаунта (из JSON, поле `client_email`) с правом **Редактор**.
5. Скопируйте ID таблицы из URL:
   `https://docs.google.com/spreadsheets/d/ЭТОТ_ID/edit`
6. Проверьте подключение:
   ```
   python check_sheets.py
   ```
   или в Telegram: `/sheets`

### На хостинге (Bothost)

Подключите репозиторий **`preudd/otchetV666`**, ветка `main`.

| Переменная | Значение |
|------------|----------|
| `BOT_TOKEN` | токен от @BotFather |
| `GOOGLE_SPREADSHEET_ID` | ID таблицы |
| `OTCHET_SA_B64_1` | первая половина base64 (из `b64_part1.txt`) |
| `OTCHET_SA_B64_2` | вторая половина base64 (из `b64_part2.txt`) |
| `DATA_DIR` | `/app/data` |

Используйте **`OTCHET_SA_B64_1` / `_2`** — новые имена, чтобы не конфликтовать со старой `GOOGLE_SERVICE_ACCOUNT_JSON_B64` на хосте.

Старые имена `GOOGLE_SERVICE_ACCOUNT_JSON_B64_1/_2` тоже поддерживаются.  
Если заданы части `_1`+`_2`, они **важнее** одной длинной `GOOGLE_SERVICE_ACCOUNT_JSON_B64`.

Сгенерировать части локально:
```
python make_b64.py service_account.json
python split_b64.py b64_for_host.txt
```

После deploy в логах: `BUILD: google-sheets-employees-v3`, `b64_len=3152`, `Google Sheets: OK`.

## Использование

1. `/start` — приветствие и меню
2. Кнопка **«Кому делаем отчёт?»** — выбор: Алина, Илья или Кира
3. `/sheets` — проверка подключения к Google Таблице
4. Отправьте файл Excel (.xlsx) с экспортом чеков

Если сотрудник не выбран заранее, бот спросит после загрузки файла.  
Кассовые поля (нал на конец дня, инкасация, остаток) бот **не заполняет**.

Дата берётся из названия файла (`Экспорт чеков от 17-01-2026.xlsx` → 17.01.2026).

## Структура Excel

- **Столбец I** — тип операции (Приход / Возврат прихода)
- **Столбец O** — список позиций чека
- **Столбец P** — итог по чеку
- **Столбец Q** — способ оплаты

## Скрипты Windows

- `openbot.bat` — запуск бота
- `close.bat` — остановка
- `restart.bat` — перезапуск

## Запись в таблицу

Бот записывает данные в блок выбранного сотрудника на листе нужного месяца:

| Сотрудник | Колонки |
|-----------|---------|
| Алина | W (дата) — AI (расчёт ДР), касса AJ–AL не заполняется |
| Илья | AN (дата) — AZ, касса BA–BC не заполняется |
| Кира | BF (дата) — BR, касса BS–BU не заполняется |
